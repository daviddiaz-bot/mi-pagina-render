Crear tu primer Pod

Un Pod es la unidad más pequeña en Kubernetes. Aquí te mostramos cómo crear uno:

pod-basico.yaml 
apiVersion: v1
kind: Pod
metadata:
  name: mi-primer-pod
  labels:
    app: mi-aplicacion
spec:
  containers:
  - name: app-container
    image: nginx:latest
    ports:
    - containerPort: 80
  restartPolicy: Always

Comandos para ejecutar:

kubectl apply -f pod-basico.yaml
kubectl get pods
kubectl describe pod mi-primer-pod

Crear un Deployment

Los Deployments manejan replicas y actualizaciones automáticamente:

deployment.yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: app-deployment
  labels:
    app: mi-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: mi-app
  template:
    metadata:
      labels:
        app: mi-app
    spec:
      containers:
      - name: app
        image: nginx:latest
        ports:
        - containerPort: 80
        resources:
          limits:
            cpu: "0.5"
            memory: "512Mi"
          requests:
            cpu: "0.25"
            memory: "256Mi"

Comandos para ejecutar:

kubectl apply -f deployment.yaml
kubectl rollout status deployment/app-deployment
kubectl scale deployment app-deployment --replicas=5

Exponer una Aplicación con Service

Los Services permiten acceder a tus aplicaciones desde dentro y fuera del cluster:

service.yaml 
apiVersion: v1
kind: Service
metadata:
  name: app-service
spec:
  type: LoadBalancer
  selector:
    app: mi-app
  ports:
  - protocol: TCP
    port: 80
    targetPort: 8080
    nodePort: 30000

Comandos para ejecutar:

kubectl apply -f service.yaml
kubectl get svc
kubectl port-forward svc/app-service 8080:80

Gestionar Configuración con ConfigMap

Los ConfigMaps almacenan datos no confidenciales en pares clave-valor:

configmap.yaml 
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  database_host: "db.example.com"
  database_port: "5432"
  log_level: "INFO"
  config.yaml: |
    server:
      port: 8080
      timeout: 30
---
apiVersion: v1
kind: Pod
metadata:
  name: app-pod
spec:
  containers:
  - name: app
    image: myapp:latest
    envFrom:
    - configMapRef:
        name: app-config

Comandos para ejecutar:

kubectl apply -f configmap.yaml
kubectl get configmap
kubectl describe configmap app-config
