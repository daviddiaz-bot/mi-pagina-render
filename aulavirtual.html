Acerca de Aula Virtual

Aula Virtual es una plataforma completa de gestión educativa que conecta estudiantes con docentes profesionales para clases personalizadas en tiempo real.

Características Principales:
  • Autenticación con 2FA
  • Sistema de pagos Stripe
  • Videollamadas Jitsi Meet
  • Gestión de clases
  • Calendario integrado
  • Sistema de reseñas
  • Panel de administración
  • API REST completa
  • Notificaciones email
  • Docker ready

Stack Tecnológico

Componente Tecnología Versión Propósito
Backend Flask 2.3+ Framework web Python
Base de Datos PostgreSQL 15+ Almacenamiento principal
Cache Redis 7+ Sesiones y colas
Web Server Nginx 1.25+ Reverse proxy
Containers Docker Latest Containerización

Guía de Implementación

Instrucciones completas para desplegar Aula Virtual desde cero

1. Requisitos Previos

Antes de comenzar: Asegúrate de tener acceso a todos los servicios necesarios antes de iniciar el despliegue.

1.1 Software Requerido

  • Docker Desktop: v20.10 o superior
  • Docker Compose: v2.0 o superior
  • Git: Para clonar el repositorio
  • Editor de texto: VS Code, Sublime, etc.

1.2 Cuentas de Servicios

Servicio Propósito Costo Link
Stripe Procesamiento de pagos Gratis (modo test) stripe.com
Gmail/SMTP Envío de emails Gratis gmail.com
AWS/DigitalOcean Hosting (producción) Variable aws.amazon.com

2. Instalación Local (Desarrollo)

1

Clonar el Repositorio

git clone https://github.com/tu-usuario/aulavirtual.git
cd aulavirtual
2

Configurar Variables de Entorno

Copia el archivo de ejemplo y edítalo con tus credenciales:

cp .env.example .env
nano .env  # o usa tu editor preferido

Variables críticas a configurar:

# Flask
SECRET_KEY=genera-con-openssl-rand-hex-32
DATABASE_URL=postgresql://aulavirtual:password@db:5432/aulavirtual

# Stripe
STRIPE_PUBLIC_KEY=pk_test_xxxxx
STRIPE_SECRET_KEY=sk_test_xxxxx
STRIPE_WEBHOOK_SECRET=whsec_xxxxx

# Email
MAIL_USERNAME=tu-email@gmail.com
MAIL_PASSWORD=contraseña-de-aplicacion

# Redis
REDIS_URL=redis://redis:6379/0
Generar SECRET_KEY: Usa openssl rand -hex 32 para generar una clave segura.
3

Iniciar con Docker

# Construir e iniciar todos los contenedores
docker-compose up --build -d

# Ver logs en tiempo real
docker-compose logs -f web
4

Inicializar Base de Datos

# Crear tablas
docker-compose exec web flask db upgrade

# Crear usuario administrador
docker-compose exec web python crear_admin.py

# Verificar instalación
docker-compose exec web python verificar_sistema.py
5

Acceder a la Aplicación

¡Listo! Abre tu navegador en http://localhost

Credenciales por defecto:

  • Email: admin@aulavirtual.com
  • Password: Admin123!

3. Configuración de Stripe

3.1 Obtener Claves de API

  1. Ve a Stripe Dashboard
  2. Crea una cuenta o inicia sesión
  3. Ve a Developers → API keys
  4. Copia las claves de Test mode:
    • Publishable key (empieza con pk_test_)
    • Secret key (empieza con sk_test_)
  5. Pégalas en tu archivo .env

3.2 Configurar Webhooks

  1. Ve a Developers → Webhooks
  2. Click en Add endpoint
  3. URL del endpoint: https://tudominio.com/webhook/stripe
  4. Selecciona eventos:
    • payment_intent.succeeded
    • payment_intent.payment_failed
    • charge.refunded
  5. Copia el Webhook signing secret a tu .env
Importante: En modo test, usa las tarjetas de prueba de Stripe. Tarjeta exitosa: 4242 4242 4242 4242

Manual de Administración

Gestión diaria del sistema, usuarios y configuraciones

1. Panel de Administración

El panel de administración está disponible en /admin y solo es accesible para usuarios con rol admin.

1.1 Acceder al Panel

  1. Inicia sesión con una cuenta de administrador
  2. En el menú superior, click en "Admin"
  3. Serás redirigido al dashboard administrativo
Roles del Sistema:
admin Acceso total al sistema
docente Puede impartir clases
estudiante Puede tomar clases

1.2 Estadísticas Generales

El dashboard muestra métricas clave del sistema:

  • Total de Usuarios: Desglosado por rol
  • Clases Completadas: Total del mes y acumulado
  • Ingresos: Totales y comisiones generadas
  • Docentes Activos: Docentes verificados y con clases
  • Tasa de Ocupación: Porcentaje de horarios ocupados

2. Gestión de Usuarios

2.1 Ver Lista de Usuarios

En Admin → Usuarios puedes:

  • Ver todos los usuarios registrados
  • Filtrar por rol, estado, fecha de registro
  • Buscar por nombre o email
  • Ver detalles completos de cada usuario

2.2 Editar Usuario

  1. Click en el botón "Editar" junto al usuario
  2. Modifica los campos necesarios:
    • Nombre completo
    • Email (único)
    • Rol del sistema
    • Estado (activo/suspendido)
  3. Click en "Guardar Cambios"
Cuidado: Cambiar el rol de un usuario puede afectar sus permisos y accesos. Asegúrate de que sea necesario.

2.3 Suspender Usuario

Para suspender temporalmente una cuenta:

  1. Ve a los detalles del usuario
  2. Click en "Suspender Cuenta"
  3. Proporciona un motivo (opcional pero recomendado)
  4. Confirma la acción

Efectos de la suspensión:

  • El usuario no puede iniciar sesión
  • Las clases programadas se cancelan automáticamente
  • Los fondos pendientes se retienen hasta resolver
  • Se envía notificación por email al usuario

2.4 Eliminar Usuario

Acción Irreversible: La eliminación es permanente y no se puede deshacer. Considera suspender en su lugar.

Para eliminar un usuario:

  1. Ve a Configuración del usuario
  2. Scroll hasta la sección "Zona de Peligro"
  3. Click en "Eliminar Usuario"
  4. Confirma escribiendo el email del usuario

3. Gestión de Docentes

3.1 Verificación de Docentes

Los docentes nuevos deben ser verificados antes de poder recibir estudiantes.

Proceso de verificación:

  1. Ve a Admin → Docentes Pendientes
  2. Revisa el perfil del docente:
    • Información personal
    • Especialidades declaradas
    • Experiencia y educación
    • Certificaciones (si las subió)
  3. Verifica credenciales y documentos
  4. Toma una decisión:
    • Aprobar: El docente puede empezar a recibir estudiantes
    • Rechazar: Proporciona motivo para rechazo
    • Solicitar más información: Envía mensaje al docente
Criterios de Aprobación:
  • Perfil completo con información veraz
  • Experiencia relevante en el área
  • Certificaciones válidas (si aplica)
  • Sin antecedentes negativos

3.2 Gestionar Docentes Activos

En Admin → Docentes puedes:

  • Ver todos los docentes verificados
  • Estadísticas por docente:
    • Número de clases impartidas
    • Calificación promedio
    • Ingresos generados
    • Tasa de cancelación
  • Suspender o eliminar docentes
  • Ver reseñas y quejas

4. Gestión Financiera

4.1 Ver Transacciones

En Admin → Finanzas → Transacciones:

  • Listado completo de todas las transacciones
  • Filtros por fecha, estado, usuario
  • Detalles de cada pago:
    • Monto total
    • Comisión de plataforma (15%)
    • Monto al docente (85%)
    • Estado del pago
    • ID de Stripe
  • Exportar a CSV para análisis

4.2 Procesar Retiros

Los docentes pueden solicitar retiros de sus ganancias.

Proceso:

  1. Ve a Admin → Finanzas → Retiros Pendientes
  2. Revisa solicitud de retiro:
    • Monto solicitado
    • Método de pago del docente
    • Datos bancarios
  3. Verifica saldo disponible del docente
  4. Procesa el pago externo (transferencia/PayPal)
  5. Marca como "Procesado" en el sistema
  6. Se envía confirmación al docente
Política de Retiros:
  • Mínimo: $50 USD
  • Procesamiento: Cada viernes
  • Tiempo: 3-5 días hábiles

4.3 Reportes Financieros

Genera reportes personalizados:

  • Ingresos por período: Diario, semanal, mensual, anual
  • Comisiones generadas: Total de ingresos de la plataforma
  • Top docentes: Los que más ganancias generan
  • Métodos de pago: Distribución por tipo de pago
  • Reembolsos: Total y motivos

Exportar reportes:

# Desde el panel admin
Admin → Reportes → Seleccionar período → Exportar CSV/PDF

5. Monitoreo del Sistema

5.1 Logs de Aplicación

# Ver logs en tiempo real
docker-compose logs -f web

# Logs de errores solamente
docker-compose logs -f web | grep ERROR

# Últimas 100 líneas
docker-compose logs --tail=100 web

5.2 Estado de Servicios

# Ver todos los contenedores
docker-compose ps

# Uso de recursos
docker stats

# Salud de PostgreSQL
docker-compose exec db pg_isready

# Ping a Redis
docker-compose exec redis redis-cli ping

5.3 Base de Datos

# Conectar a PostgreSQL
docker-compose exec db psql -U aulavirtual -d aulavirtual

# Ver tamaño de la BD
\l+

# Ver tablas
\dt

# Contar usuarios
SELECT rol, COUNT(*) FROM usuario GROUP BY rol;

5.4 Backups

Backup de Base de Datos:

# Crear backup
docker-compose exec db pg_dump -U aulavirtual aulavirtual > backup_$(date +%Y%m%d).sql

# Restaurar backup
docker-compose exec -T db psql -U aulavirtual aulavirtual < backup_20260303.sql
Backups Automáticos: Configura un cron job para backups diarios y almacena en S3/cloud storage.

Manual del Estudiante

Guía completa para estudiantes de Aula Virtual

1. Comenzar en Aula Virtual

1.1 Crear una Cuenta

  1. Ve a la página principal
  2. Click en "Registrarse"
  3. Completa el formulario:
    • Nombre completo
    • Email (debe ser único)
    • Contraseña segura (mínimo 8 caracteres)
    • Selecciona: Estudiante
  4. Acepta términos y condiciones
  5. Click en "Crear Cuenta"
  6. Recibirás un email de confirmación
¡Bienvenido! Tu cuenta está lista. Ahora puedes comenzar a buscar docentes.

1.2 Completar tu Perfil

Es recomendable completar tu perfil para una mejor experiencia:

  1. Ve a Profile → Configuración
  2. Agrega una foto de perfil
  3. Completa tu biografía
  4. Agrega número de teléfono (opcional)

2. Buscar y Seleccionar Docentes

2.1 Explorar Docentes

En "Buscar Docentes" puedes:

  • Ver todos los docentes disponibles
  • Filtrar por:
    • Especialidad (Matemáticas, Física, Inglés, etc.)
    • Rango de precios
    • Calificación mínima
  • Ordenar por:
    • Calificación (mayor a menor)
    • Precio (menor a mayor)
    • Experiencia

2.2 Ver Perfil del Docente

Click en cualquier docente para ver:

  • Información básica: Foto, nombre, especialidades
  • Experiencia: Años de experiencia, educación
  • Tarifa: Precio por hora
  • Calificación: Promedio y número de reseñas
  • Reseñas: Comentarios de otros estudiantes
  • Disponibilidad: Horarios disponibles
Tip: Lee las reseñas de otros estudiantes para conocer la experiencia de aprendizaje con cada docente.

3. Reservar una Clase

3.1 Proceso de Reserva

  1. Seleccionar fecha y hora:
    • Ve al calendario del docente
    • Horarios disponibles en verde
    • Click en el horario deseado
  2. Confirmar detalles:
    • Fecha y hora
    • Duración (normalmente 60 minutos)
    • Precio total
  3. Agregar nota (opcional):
    • Describe qué quieres aprender
    • Menciona dificultades específicas
    • Solicita materiales previos
  4. Proceder al pago

3.2 Realizar el Pago

El pago se procesa de forma segura con Stripe:

  1. Selecciona método de pago:
    • Tarjeta de crédito/débito
    • PayPal (si está habilitado)
  2. Ingresa datos de la tarjeta:
    • Número de tarjeta
    • Fecha de vencimiento
    • CVC
  3. Verifica el monto
  4. Click en "Pagar"
  5. Confirmación instantánea
Pago Seguro: Usamos Stripe, líder mundial en procesamiento de pagos. Tus datos están protegidos con encriptación de nivel bancario.

3.3 Confirmación

Después del pago:

  • ✅ Recibes email de confirmación
  • ✅ La clase aparece en tu calendario
  • ✅ El docente recibe notificación
  • ✅ Recordatorio 1 hora antes de la clase
  • ✅ Notificación 15 minutos antes para unirse

4. Tomar la Clase

4.1 Preparación

Antes de la clase:

  • ✅ Verifica tu conexión a internet
  • ✅ Prueba tu cámara y micrófono
  • ✅ Ten a mano materiales necesarios
  • ✅ Encuentra un lugar tranquilo
  • ✅ Cierra aplicaciones que consuman ancho de banda

4.2 Unirse a la Videollamada

15 minutos antes de la clase:

  1. Ve a "Mi Calendario"
  2. Verás el botón "Unirse a Clase" activo
  3. Click en el botón
  4. Se abre una nueva ventana con Jitsi Meet
  5. Permite acceso a cámara y micrófono
  6. Espera a que el docente se una
Sistema de Video: Usamos Jitsi Meet, no necesitas instalar nada. Funciona en cualquier navegador moderno.

4.3 Durante la Clase

Controles de la videollamada:

  • Micrófono: Activar/silenciar audio
  • Cámara: Activar/desactivar video
  • Compartir pantalla: Mostrar tu pantalla
  • Chat: Enviar mensajes de texto
  • Levantar mano: Solicitar turno
  • Configuración: Ajustes de audio/video

Buenas prácticas:

  • Sé puntual
  • Mantén tu cámara encendida
  • Silencia tu micrófono cuando no hables
  • Participa activamente
  • Haz preguntas
  • Toma notas

4.4 Finalizar la Clase

Al terminar:

  1. Agradece al docente
  2. Confirma que entendiste todo
  3. Click en "Salir" para abandonar la videollamada
  4. La clase se marca automáticamente como completada

5. Después de la Clase

5.1 Dejar una Reseña

Es importante calificar al docente:

  1. Recibirás una notificación para dejar reseña
  2. O ve a "Mis Clases" → Click en "Dejar Reseña"
  3. Califica de 1 a 5 estrellas:
    • ⭐ Muy malo
    • ⭐⭐ Malo
    • ⭐⭐⭐ Regular
    • ⭐⭐⭐⭐ Bueno
    • ⭐⭐⭐⭐⭐ Excelente
  4. Escribe un comentario (opcional pero recomendado)
  5. Click en "Enviar Reseña"
Ayuda a la comunidad: Tus reseñas honestas ayudan a otros estudiantes a elegir buenos docentes.

5.2 Acceder a Materiales

Si el docente compartió materiales:

  1. Ve a "Mis Clases"
  2. Click en la clase completada
  3. Sección "Materiales"
  4. Descarga PDFs, documentos, links, etc.

5.3 Programar Nueva Clase

Si te gustó el docente:

  • Click en "Reservar Otra Clase" desde el perfil
  • O marca como "Favorito" para encontrarlo fácilmente

6. Política de Cancelación

Tiempo antes de la clase Reembolso Penalización
Más de 24 horas 100% Ninguna
Entre 12-24 horas 50% Ninguna
Menos de 12 horas 0% No se reembolsa
Docente no se presenta 100% + crédito Ninguna para estudiante

Cómo Cancelar

  1. Ve a "Mi Calendario"
  2. Busca la clase que deseas cancelar
  3. Click en "Cancelar Clase"
  4. Confirma la cancelación
  5. El reembolso se procesa automáticamente según la política
Importante: Cancela con la mayor anticipación posible para evitar pérdida de dinero y permitir que el docente reorganice su agenda.

7. Preguntas Frecuentes (FAQ)

Los pagos se procesan de forma segura con Stripe. Aceptamos tarjetas de crédito y débito. El cobro se realiza al momento de la reserva. Si cancelas con anticipación suficiente, el reembolso es automático.

Si tienes problemas para unirte a la videollamada, primero verifica tu conexión a internet. Si el problema persiste, contacta a soporte inmediatamente. Si los problemas técnicos impiden la clase, puedes solicitar reembolso completo o reprogramación.

Solo puedes grabar las clases con el consentimiento expreso del docente. Es importante respetar los derechos de privacidad y propiedad intelectual. Pregunta al docente antes de la clase si permite grabación.

Primero, intenta resolver el problema directamente con el docente. Si no es posible, contacta a soporte dentro de las 24 horas posteriores a la clase. Revisaremos el caso y evaluaremos si corresponde un reembolso parcial o total.

Manual del Docente

Guía completa para docentes de Aula Virtual

1. Empezar como Docente

1.1 Registrarse

  1. Ve a la página principal
  2. Click en "Registrarse"
  3. Selecciona rol: Docente
  4. Completa el formulario básico
  5. Verifica tu email

1.2 Completar Perfil Profesional

Después del registro, debes completar tu perfil:

Importante: No podrás recibir estudiantes hasta que tu perfil sea verificado por el administrador.

Información requerida:

  • Foto de perfil: Profesional, clara, actual
  • Especialidades: Máximo 3-5 áreas de especialización
  • Descripción: Resume tu experiencia y método de enseñanza (200-500 palabras)
  • Experiencia: Años enseñando, cargos previos
  • Educación: Títulos, certificaciones, cursos
  • Precio por hora: Tarifa en USD (puedes cambiarla después)

1.3 Verificación

El proceso de verificación toma 1-3 días hábiles:

  1. El equipo admin revisa tu perfil
  2. Verifican tus credenciales
  3. Pueden solicitar documentos adicionales
  4. Recibes notificación de aprobación o rechazo
  5. Si apruebas, ¡ya puedes empezar a recibir estudiantes!
Tips para aprobación rápida:
  • Completa todos los campos requeridos
  • Usa una foto profesional
  • Describe claramente tu experiencia
  • Adjunta certificados si los tienes

2. Configurar Disponibilidad

2.1 Crear Horarios

Ve a "Calendario" → "Configurar Disponibilidad":

  1. Selecciona días de la semana
  2. Define rangos horarios (ej: Lunes 9:00-17:00)
  3. Puedes crear múltiples bloques por día
  4. Los estudiantes solo podrán reservar en estos horarios

Ejemplo de configuración:

Lunes: 09:00-12:00, 14:00-18:00
Martes: 10:00-13:00, 15:00-19:00
Miércoles: Día libre
Jueves: 09:00-17:00 (corrido)
Viernes: 08:00-12:00
Sábado: 10:00-14:00
Domingo: Cerrado

2.2 Bloquear Fechas Específicas

Para vacaciones o días no disponibles:

  1. Ve al calendario
  2. Click en el día que quieres bloquear
  3. Selecciona "Bloquear día completo"
  4. O bloquea solo ciertas horas
Flexibilidad: Puedes cambiar tu disponibilidad en cualquier momento, pero respeta las clases ya reservadas.

3. Gestionar Clases

3.1 Recibir Reservas

Cuando un estudiante reserva:

  • ✅ Recibes email de notificación
  • ✅ Aparece en tu calendario
  • ✅ Ves información del estudiante
  • ✅ Puedes revisar su nota/mensaje

3.2 Preparar la Clase

Antes de la clase:

  1. Lee la nota del estudiante (si la dejó)
  2. Prepara materiales relevantes
  3. Planifica la estructura de la clase
  4. Prueba tu equipo (cámara, micrófono, internet)

3.3 Impartir la Clase

15 minutos antes:

  1. Ve a "Mi Calendario"
  2. Click en "Unirse a Clase"
  3. Se abre Jitsi Meet
  4. Espera al estudiante
  5. Comienza puntualmente

Durante la clase:

  • Sé puntual y profesional
  • Usa la pizarra virtual si es necesario
  • Comparte pantalla para mostrar materiales
  • Mantén una actitud positiva
  • Adapta tu ritmo al estudiante
  • Deja tiempo para preguntas
Tips para clases efectivas:
  • Pregunta qué quiere lograr el estudiante
  • Explica con ejemplos prácticos
  • Verifica comprensión frecuentemente
  • Asigna pequeñas tareas si es apropiado
  • Resume los puntos clave al final

3.4 Después de la Clase

  1. La clase se marca automáticamente como completada
  2. Puedes subir materiales adicionales
  3. El pago se libera después de 24-48 horas
  4. El estudiante puede dejarte una reseña

4. Gestión Financiera

4.1 Tus Ganancias

Estructura de comisiones:

  • Estudiante paga: $50 (ejemplo)
  • Comisión de plataforma: -$7.50 (15%)
  • Tú recibes: $42.50 (85%)
Ganancias: Recibes el 85% de cada clase. La comisión del 15% cubre costos de plataforma, pagos, soporte y marketing.

4.2 Panel de Finanzas

En "Finanzas" puedes ver:

  • Saldo Disponible: Dinero listo para retirar
  • Saldo Pendiente: De clases completadas hace menos de 48h
  • Total Ingresos: Histórico completo
  • Ingresos del Mes: Ganancias actuales
  • Transacciones: Historial detallado
  • Retiros: Historial de pagos recibidos

4.3 Solicitar Retiro

Requisitos:

  • Mínimo: $50 USD
  • Frecuencia: 1 vez por semana
  • Método configurado previamente

Proceso:

  1. Ve a "Finanzas" → "Solicitar Retiro"
  2. Verifica tu método de pago configurado
  3. Ingresa el monto a retirar
  4. Confirma la solicitud
  5. Procesamiento: próximo viernes
  6. Recibirás el pago en 3-5 días hábiles

4.4 Configurar Método de Pago

En "Configuración" → "Métodos de Pago":

Opciones disponibles:

  • Transferencia Bancaria:
    • Nombre del banco
    • Número de cuenta
    • Nombre del titular
    • Identificación fiscal
  • PayPal:
    • Email de PayPal
Importante: Verifica que tus datos de pago sean correctos. Errores pueden causar demoras en los retiros.

5. Construir tu Reputación

5.1 Sistema de Calificaciones

1. Autenticación y Autorización

1.1 Sistema de Autenticación

Hashing de Contraseñas

Las contraseñas se almacenan usando Werkzeug PBKDF2-SHA256 con salts automáticos:

# En models.py
from werkzeug.security import generate_password_hash, check_password_hash

def set_password(self, password):
    """Establecer contraseña hasheada"""
    self.password_hash = generate_password_hash(password)

def check_password(self, password):
    """Verificar contraseña"""
    return check_password_hash(self.password_hash, password)
Seguro: PBKDF2 con 260,000 iteraciones + salt único por contraseña.

Política de Contraseñas

  • Mínimo 8 caracteres
  • No se almacena texto plano
  • Tokens de recuperación expiran en 1 hora
  • Verificación por email obligatoria

1.2 Autenticación de Dos Factores (2FA)

Implementación completa de 2FA usando TOTP (Time-based One-Time Password):

# Configuración 2FA
import pyotp
import qrcode

def generate_totp_secret(self):
    """Generar secreto para 2FA"""
    if not self.totp_secret:
        self.totp_secret = base64.b32encode(os.urandom(10)).decode('utf-8')
    return self.totp_secret

def verify_totp(self, token):
    """Verificar token TOTP"""
    totp = pyotp.TOTP(self.totp_secret)
    return totp.verify(token, valid_window=1)

def generate_backup_codes(self):
    """Generar 10 códigos de respaldo"""
    codes = []
    for _ in range(10):
        code = secrets.token_hex(4).upper()
        codes.append(code)
    self.backup_codes = ','.join([generate_password_hash(c) for c in codes])
    return codes

Aplicaciones compatibles Google Authenticator, Authy, Microsoft Authenticator
Códigos de respaldo 10 códigos únicos, un solo uso
Ventana de validez ±30 segundos (valid_window=1)
QR Code Generado automáticamente para easy setup

2.2 Tokens de Recuperación

# Generación de token seguro
from itsdangerous import URLSafeTimedSerializer

def generate_reset_token(usuario):
    """Generar token para resetear contraseña"""
    serializer = URLSafeTimedSerializer(current_app.config['SECRET_KEY'])
    return serializer.dumps(usuario.email, salt='password-reset-salt')

def verify_reset_token(token, expiration=3600):
    """Verificar token (expira en 1 hora)"""
    serializer = URLSafeTimedSerializer(current_app.config['SECRET_KEY'])
    try:
        email = serializer.loads(token, salt='password-reset-salt', max_age=expiration)
    except:
        return None
    return Usuario.query.filter_by(email=email).first()
Importante: Los tokens de recuperación expiran en 1 hora por seguridad.

print(f"Especialidad: {docente['especialidad']}")
print(f"Precio/hora: ${docente['precio_hora']}")

6.2 JavaScript (Node.js)

const axios = require('axios');

const API_KEY = 'tu-api-key';
const BASE_URL = 'https://aulavirtual.com/api';

const headers = {
    'X-API-Key': API_KEY
};

// Listar docentes
async function listarDocentes() {
    try {
        const response = await axios.get(`${BASE_URL}/docentes`, { headers });
        console.log(response.data.docentes);
    } catch (error) {
        console.error('Error:', error.response.data);
    }
}

// Obtener clase por ID
async function obtenerClase(claseId) {
    try {
        const response = await axios.get(`${BASE_URL}/clases/${claseId}`, { headers });
        return response.data;
    } catch (error) {
        console.error('Error:', error.response.data);
    }
}

listarDocentes();

1. Preguntas Generales

¿Qué es Aula Virtual?

Aula Virtual es una plataforma educativa que conecta estudiantes con docentes calificados para clases personalizadas en línea. Ofrecemos una amplia variedad de materias y horarios flexibles adaptados a tus necesidades.

¿Cómo funciona la plataforma?

  1. Crea una cuenta gratuita
  2. Busca docentes según tu materia de interés
  3. Solicita una clase en el horario que prefieras
  4. Realiza el pago de forma segura con Stripe
  5. Únete a la clase mediante videoconferencia (Jitsi Meet)

¿Es gratis usar Aula Virtual?

Crear una cuenta y buscar docentes es completamente gratis. Solo pagas por las clases que solicites al precio fijado por cada docente (típicamente $15-50 USD por hora).

¿Qué necesito para tomar clases?

  • Conexión a internet estable
  • Computadora, tablet o smartphone
  • Navegador web moderno (Chrome, Firefox, Safari, Edge)
  • Cámara y micrófono (recomendado)
  • Auriculares (opcional, pero mejora la calidad del audio)

¿En qué idiomas está disponible la plataforma?

Actualmente Aula Virtual está disponible en español. Las clases pueden impartirse en diferentes idiomas según la preferencia del docente y estudiante.

2. Para Estudiantes

¿Cómo creo una cuenta?

  1. Haz clic en "Registrarse" en la página principal
  2. Completa el formulario con tu nombre, email y contraseña
  3. Verifica tu email (revisa la bandeja de spam si no lo recibes)
  4. Configura la autenticación de dos factores (2FA) para mayor seguridad

¿Cómo busco un docente?

Desde tu dashboard, ve a "Buscar Docentes" y filtra por:

  • Especialidad: Matemáticas, Física, Inglés, Programación, etc.
  • Precio: Define tu rango de presupuesto
  • Calificación: Revisa reseñas de otros estudiantes

Cada perfil muestra experiencia, educación, precios y reseñas.

¿Cómo reservo una clase?

  1. Selecciona el docente que deseas
  2. Haz clic en "Solicitar Clase"
  3. Completa el formulario:
    • Título de la clase
    • Descripción (temas a cubrir)
    • Fecha y hora
    • Duración (mínimo 30 minutos)
  4. Procede con el pago

¿Cómo pago las clases?

Aceptamos pagos a través de Stripe, que procesa de forma segura:

  • Tarjetas de crédito (Visa, Mastercard, American Express)
  • Tarjetas de débito

Tu información financiera nunca se almacena en nuestros servidores.

Seguro: Todas las transacciones están encriptadas con SSL/TLS.

¿Cómo me conecto a una clase?

  1. Una vez confirmado el pago, recibirás un email de confirmación
  2. En "Mis Clases" verás el link de la videoconferencia (Jitsi Meet)
  3. 5 minutos antes de la clase, haz clic en el link
  4. Autoriza el uso de cámara y micrófono en tu navegador
  5. ¡Listo! Ya estás en la clase
Nota: No necesitas descargar ningún software. Jitsi funciona directamente en tu navegador.

¿Puedo cancelar una clase?

Sí, puedes cancelar desde "Mis Clases". Las políticas de reembolso son:

Cancelación Reembolso
Más de 24 horas antes 100%
12-24 horas antes 50%
Menos de 12 horas antes 0%

¿Cómo dejo una reseña?

Después de completar una clase:

  1. Ve a "Mis Clases"
  2. Selecciona la clase completada
  3. Haz clic en "Dejar Reseña"
  4. Califica de 1 a 5 estrellas
  5. Escribe un comentario (opcional)

Las reseñas ayudan a otros estudiantes a elegir docentes.

  • Credenciales educativas
  • Experiencia profesional
  • Calidad del perfil

Cuando un estudiante solicite una clase:

4. Aspectos Técnicos

¿Qué tecnologías usa Aula Virtual?

Nuestra plataforma está construida con tecnologías modernas y seguras:

Componente Tecnología
Backend Python + Flask
Base de datos PostgreSQL
Pagos Stripe
Videoconferencia Jitsi Meet
Autenticación Flask-Login + 2FA (TOTP)
Tareas asíncronas Celery + Redis

¿Es segura la plataforma?

Sí, implementamos múltiples capas de seguridad:

  • HTTPS/SSL: Todas las conexiones están encriptadas
  • 2FA: Autenticación de dos factores obligatoria
  • Hashing de contraseñas: PBKDF2 con salt
  • Protecciones: SQL Injection, XSS, CSRF
  • Rate limiting: Prevención de ataques por fuerza bruta
  • Backups: Diarios y encriptados

Ver sección Seguridad para más detalles.

¿Hay una app móvil?

Actualmente Aula Virtual es una aplicación web 100% responsive que funciona perfectamente en cualquier dispositivo móvil a través del navegador. Estamos evaluando desarrollar aplicaciones nativas para iOS y Android en el futuro.

¿Hay una API disponible?

Sí, ofrecemos una API REST para integraciones. Ver la sección API Reference para documentación completa.

Endpoints principales:

  • GET /api/docentes - Listar docentes
  • GET /api/docentes/{id} - Detalles de docente
  • GET /api/clases - Listar clases
  • GET /api/clases/{id} - Detalles de clase
API Key: Solicita tu API key a soporte@aulavirtual.com

Problema Solución
No puedo conectarme a la videollamada Verifica que tu navegador tenga permisos para cámara y micrófono. Recarga la página.
