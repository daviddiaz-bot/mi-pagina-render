📋 Flujo de Usuario Completo (Ejemplo):
  1. Usuario: Ingresa email "usuario161@correo.co" en modal
  2. Frontend: Valida email y muestra interfaz de chat
  3. Usuario: Escribe mensaje "¿Cuál es el estado de mi ticket?"
  4. Backend: Transmite {email: "usuario161@correo.co", message: "¿Cuál es..."} a n8n
  5. n8n Webhook: Recibe solicitud
  6. PostgreSQL: Ejecuta LEFT JOIN → Retorna 2 tickets del cliente:
  7. Ticket #664: "Problema de login", estado: "Abierto", prioridad: "Alta"
  8. Ticket #1048: "Consulta de factura", estado: "Resuelto", prioridad: "Media"
  9. Preparar Contexto: Formatea datos → "Cliente: Juan García, Email: usuario161@correo.co, Tiene 2 tickets: #664 (Abierto, Alta)..."
  10. Google Gemini: Procesa contexto + mensaje → Genera: "Hola Juan, veo que tienes 2 tickets. El #664 sigue abierto con prioridad alta. Te ayudaré..."
  11. n8n Response: Retorna JSON con respuesta
  12. Backend: Recibe respuesta y la envía al frontend
  13. Frontend: Muestra respuesta en chat en tiempo real

2.3 Preparar Contexto (JavaScript Node)

// Obtener TODOS los items (no solo el primero) const items = $input.all(); let dbContext = ""; if (items.length > 0 && items[0].json.ticket_id) { // Hay tickets para este cliente items.forEach((item, i) => { const ticket = item.json; dbContext += `Ticket #${ticket.ticket_id}: ${ticket.asunto} (${ticket.estado}, Prioridad: ${ticket.prioridad})\n`; }); } else if (items.length > 0) { // Cliente existe pero sin tickets const cliente = items[0].json; dbContext = `Cliente: ${cliente.cliente} (${cliente.email}) - Sin tickets abiertos`; } else { dbContext = "Cliente no encontrado en la base de datos"; } return { json: { dbContext: dbContext } };

🔧 Solución de Problemas

❌ Error: "Connection refused" en n8n-service ▼

Causa: Backend no puede conectarse a n8n en 192.168.1.12:5678

# Verificar que n8n está corriendo ping 192.168.1.12 # Verificar puerto 5678 abierto telnet 192.168.1.12 5678 # Verificar URL en .env grep N8N_WEBHOOK_URL chat-app-backend/.env # Debe ser HTTP (no HTTPS) en desarrollo # http://192.168.1.12:5678/webhook/ai-agent-chat-sql
❌ Error: "relation 'clientes' does not exist" ▼

Causa: Las tablas de PostgreSQL no existen

# Conectarse a PostgreSQL psql -h 192.168.1.6 -p 5433 -U tu_usuario -d customer_service # Crear tablas CREATE TABLE clientes ( id SERIAL PRIMARY KEY, nombre VARCHAR(255) NOT NULL, email VARCHAR(255) UNIQUE NOT NULL ); CREATE TABLE tickets ( id SERIAL PRIMARY KEY, cliente_id INTEGER REFERENCES clientes(id), asunto VARCHAR(255), estado VARCHAR(50), prioridad VARCHAR(20) ); # Insertar datos de prueba INSERT INTO clientes (nombre, email) VALUES ('Juan García', 'usuario161@correo.co'); INSERT INTO tickets (cliente_id, asunto, estado, prioridad) VALUES (1, 'Test Ticket', 'Abierto', 'Alta');
❌ Error: "Preparar Contexto - Cannot read property 'ticket_id'" ▼

Causa: Usar $input.first() en lugar de $input.all()

# INCORRECTO: const item = $input.first().json; // Solo devuelve 1 item # CORRECTO: const items = $input.all(); // Devuelve todos los items
