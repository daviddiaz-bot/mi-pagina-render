Introducción

¿Qué es Isard VDI y por qué usarlo?

¿Qué es Isard VDI?

Isard VDI es una solución de escritorio virtual (VDI - Virtual Desktop Infrastructure) de código abierto diseñada para proporcionar entornos de escritorio virtuales a usuarios finales. Es especialmente adecuada para entornos educativos, aunque también puede utilizarse en entornos empresariales.

ℹ️
Información Importante

Isard VDI permite gestionar múltiples escritorios virtuales desde una interfaz web centralizada, facilitando la administración y el acceso a recursos informáticos.

🎓

Entorno Educativo

Ideal para aulas de informática, laboratorios y centros educativos que necesitan gestionar múltiples escritorios.

🔓

Código Abierto

Totalmente open source bajo licencia AGPL-3.0, permitiendo personalización y adaptación a necesidades específicas.

🌐

Acceso Web

Acceso a escritorios virtuales desde cualquier navegador web, sin necesidad de clientes adicionales.

Características Principales

  • Gestión Centralizada: Panel de administración web para gestionar usuarios, escritorios y recursos.
  • Múltiples Hipervisores: Soporte para KVM, VMware, VirtualBox y más.
  • Plantillas: Creación y gestión de plantillas de escritorio para despliegues rápidos.
  • Autenticación Flexible: Integración con LDAP, Active Directory y otros sistemas de autenticación.
  • API REST: API completa para integración con sistemas externos.
  • Despliegue Flexible: Soporte para Docker, Docker Compose y Kubernetes.

Instalación

Guía paso a paso para desplegar Isard VDI

Instalación con Docker

El método más rápido para desplegar Isard VDI es usando Docker directamente.

Instalar Docker

Primero, asegúrese de tener Docker instalado en su sistema:

# Actualizar el sistema sudo apt update && sudo apt upgrade -y # Instalar dependencias sudo apt install -y apt-transport-https ca-certificates curl software-properties-common # Añadir la clave GPG de Docker curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg # Añadir el repositorio de Docker echo "deb [arch=amd64 signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null # Instalar Docker sudo apt update sudo apt install -y docker-ce docker-ce-cli containerd.io # Añadir usuario al grupo docker sudo usermod -aG docker $USER # Verificar instalación docker --version
Clonar el Repositorio

Clone el repositorio oficial de Isard VDI:

# Clonar el repositorio git clone https://gitlab.com/isard/isardvdi.git cd isardvdi
Configurar Variables de Entorno

Cree y configure el archivo de variables de entorno:

# Copiar el archivo de ejemplo cp .env.example .env # Editar el archivo con su editor preferido nano .env

Configure las siguientes variables importantes:

# Dominio de la aplicación DOMAIN=isardvmi.tu-dominio.com # Configuración de correo SMTP_HOST=smtp.gmail.com SMTP_PORT=587 SMTP_USER=tu-email@gmail.com SMTP_PASSWORD=tu-contraseña # Configuración de base de datos DB_HOST=postgres DB_NAME=isardvdi DB_USER=isardvdi DB_PASSWORD=tu-contraseña-segura # Secret key para Django SECRET_KEY=genera-una-clave-segura-aqui
Iniciar los Contenedores

Inicie todos los servicios de Isard VDI:

# Construir e iniciar los contenedores docker-compose up -d # Verificar el estado de los contenedores docker-compose ps # Ver los logs docker-compose logs -f
Acceder a Isard VDI

Una vez iniciados los contenedores, acceda a Isard VDI:

  • URL: https://isardvmi.tu-dominio.com
  • Usuario admin: admin
  • Contraseña admin: IsardVDI (cámbiela inmediatamente)
✅
¡Instalación Completada!

Isard VDI está ahora listo para usar. Acceda a la interfaz web y comience a configurar su entorno VDI.

Instalación con Docker Compose

Para despliegues más complejos, Docker Compose ofrece mayor flexibilidad en la configuración.

Instalar Docker Compose
# Descargar Docker Compose sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose # Dar permisos de ejecución sudo chmod +x /usr/local/bin/docker-compose # Verificar instalación docker-compose --version
Crear docker-compose.yml

Cree un archivo docker-compose.yml personalizado:

version: '3.8' services: isardvdi: image: isard/isardvdi:latest container_name: isardvdi restart: unless-stopped ports: - "80:80" - "443:443" volumes: - ./data:/data - ./media:/media environment: - DOMAIN=isardvdi.local - SECRET_KEY=your-secret-key-here depends_on: - postgres - redis postgres: image: postgres:14 container_name: isardvdi-postgres restart: unless-stopped volumes: - ./postgres-data:/var/lib/postgresql/data environment: - POSTGRES_DB=isardvdi - POSTGRES_USER=isardvdi - POSTGRES_PASSWORD=your-password redis: image: redis:7-alpine container_name: isardvdi-redis restart: unless-stopped volumes: - ./redis-data:/data volumes: postgres-data: redis-data:
Iniciar Servicios
# Iniciar todos los servicios docker-compose up -d # Verificar estado docker-compose ps # Ver logs docker-compose logs -f isardvdi

Instalación con Kubernetes

Para despliegues en producción a gran escala, Kubernetes es la opción recomendada.

Instalar Helm
# Descargar e instalar Helm curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash # Verificar instalación helm version
Añadir el Repositorio de Helm
# Añadir el repositorio de Isard VDI helm repo add isardvdi https://isard.gitlab.io/isardvdi-charts # Actualizar repositorios helm repo update
Crear values.yaml

Cree un archivo values.yaml con su configuración:

replicaCount: 2 image: repository: isard/isardvdi tag: latest pullPolicy: Always service: type: LoadBalancer port: 443 ingress: enabled: true className: nginx annotations: cert-manager.io/cluster-issuer: letsencrypt-prod hosts: - host: isardvdi.example.com paths: - path: / pathType: Prefix tls: - secretName: isardvdi-tls hosts: - isardvdi.example.com resources: limits: cpu: "2000m" memory: "4Gi" requests: cpu: "500m" memory: "1Gi" postgresql: enabled: true auth: password: your-secure-password redis: enabled: true
Desplegar con Helm
# Crear namespace kubectl create namespace isardvdi # Desplegar Isard VDI helm install isardvdi isardvdi/isardvdi -n isardvdi -f values.yaml # Verificar el despliegue kubectl get pods -n isardvdi # Ver logs kubectl logs -n isardvdi -l app.kubernetes.io/name=isardvdi -f

Configuración

Ajustes y personalización de Isard VDI

Configuración Inicial

Después de la instalación, realice la configuración inicial desde el panel de administración.

Acceder al Panel de Administración

Acceda a https://tu-dominio/admin con las credenciales de administrador.

Configurar Dominio y SSL

Configure el dominio y los certificados SSL para conexiones seguras:

# Usar Let's Encrypt para certificados SSL gratuitos certbot --nginx -d isardvdi.tu-dominio.com
Configurar Correo Electrónico

Configure el servidor SMTP para notificaciones:

# Editar configuración de correo nano /etc/isardvdi/settings.py # Configurar SMTP EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend' EMAIL_HOST = 'smtp.gmail.com' EMAIL_PORT = 587 EMAIL_USE_TLS = True EMAIL_HOST_USER = 'tu-email@gmail.com' EMAIL_HOST_PASSWORD = 'tu-contraseña'

Configuración de Almacenamiento

Configure los diferentes tipos de almacenamiento para escritorios y plantillas:

Almacenamiento Local

Para despliegues pequeños, use almacenamiento local:

storage: type: local path: /var/lib/isardvdi/storage

Almacenamiento NFS

Para despliegues distribuidos, use NFS:

storage: type: nfs server: 192.168.1.100 path: /export/isardvdi

Configuración de Rendimiento

Optimice el rendimiento según sus necesidades:

Parámetro Descripción Valor Recomendado
MAX_CONCURRENT_DESKTOPS Máximo de escritorios concurrentes 50-100
DESKTOP_TIMEOUT Timeout de inactividad (minutos) 30
ALLOWED_DOMAINS Dominios permitidos para registro tu-dominio.com
DESKTOP_DEFAULT_RAM RAM por defecto (MB) 2048
DESKTOP_DEFAULT_VCPUS vCPUs por defecto 2

Configurar KVM

Para configurar KVM como hipervisor:

Instalar KVM
# Instalar paquetes KVM sudo apt install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils virtinst virt-manager # Verificar soporte de virtualización kvm-ok # Añadir usuario al grupo libvirt sudo usermod -aG libvirt $USER
Configurar Libvirt
# Habilitar y iniciar libvirtd sudo systemctl enable libvirtd sudo systemctl start libvirtd # Configurar libvirt para escucha TCP sudo nano /etc/libvirt/libvirtd.conf # Añadir estas líneas: listen_tls = 0 listen_tcp = 1 tcp_port = "16509" listen_addr = "0.0.0.0" auth_tcp = "none"
Configurar Isard VDI

Configure Isard VDI para usar KVM:

hypervisor: type: kvm connection: qemu+tcp://localhost:16509/system storage_pool: default network: default

Configurar VMware

Para integrar con VMware ESXi:

hypervisor: type: vmware host: esxi.example.com port: 443 username: root password: your-password datacenter: ha-datacenter datastore: datastore1

Gestión de Plantillas

Creación y gestión de plantillas de escritorio

¿Qué son las Plantillas?

Las plantillas son imágenes base de escritorio virtual que se pueden clonar para crear nuevos escritorios de forma rápida y consistente.

💡
Ventajas de usar Plantillas

Las plantillas permiten desplegar escritorios en segundos, garantizando que todos los usuarios tengan el mismo entorno configurado.

Crear una Plantilla

Crear una Máquina Virtual Base

Cree una VM con el sistema operativo deseado:

# Crear VM con virt-install virt-install \ --name template-ubuntu \ --ram 2048 \ --vcpus 2 \ --disk path=/var/lib/libvirt/images/template-ubuntu.qcow2,size=20 \ --cdrom /path/to/ubuntu-22.04.iso \ --network network=default \ --graphics spice
Instalar y Configurar el SO

Instale el sistema operativo y configure:

  • Actualizaciones del sistema
  • Software necesario (oficina, navegadores, etc.)
  • Configuraciones de red
  • Agentes de Isard VDI
Sysprep (Windows) o Cloud-Init (Linux)

Prepare la plantilla para clonación:

# Para Linux - instalar cloud-init sudo apt install cloud-init # Limpiar máquina sudo virt-sysprep -d template-ubuntu # Apagar la VM sudo virsh shutdown template-ubuntu
Importar en Isard VDI

Importe la plantilla en Isard VDI:

# Usar la API o interfaz web para importar curl -X POST https://isardvdi.local/api/templates \ -H "Authorization: Bearer YOUR_TOKEN" \ -F "file=@/var/lib/libvirt/images/template-ubuntu.qcow2" \ -F "name=Ubuntu 22.04 Template" \ -F "description=Plantilla base de Ubuntu 22.04"

Configurar LDAP/Active Directory

Configurar Conexión LDAP
# settings.py AUTHENTICATION_BACKENDS = [ 'django.contrib.auth.backends.ModelBackend', 'isardvdi.authentication.LDAPBackend', ] LDAP_SERVER = 'ldap://ad.example.com' LDAP_PORT = 389 LDAP_BASE_DN = 'DC=example,DC=com' LDAP_USER_SEARCH = '(sAMAccountName={username})' LDAP_BIND_DN = 'CN=admin,DC=example,DC=com' LDAP_BIND_PASSWORD = 'your-password'

API REST

Integración y automatización con la API

Introducción a la API

Isard VDI expone una API REST completa para integración con sistemas externos y automatización de tareas.

📚
Documentación de la API

La documentación completa de la API está disponible en https://tu-dominio/api/docs

Autenticación

La API usa tokens JWT para autenticación:

# Obtener token curl -X POST https://isardvdi.local/api/auth/token \ -H "Content-Type: application/json" \ -d '{"username": "admin", "password": "your-password"}' # Respuesta { "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." } # Usar token en solicitudes curl -X GET https://isardvdi.local/api/desktops \ -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."

Endpoints Principales

Escritorios (Desktops) ▼
# Listar escritorios GET /api/desktops # Obtener detalles de un escritorio GET /api/desktops/{id} # Crear escritorio POST /api/desktops { "name": "Mi Escritorio", "template": "ubuntu-22.04", "ram": 2048, "vcpus": 2 } # Iniciar escritorio POST /api/desktops/{id}/start # Detener escritorio POST /api/desktops/{id}/stop # Eliminar escritorio DELETE /api/desktops/{id}
Plantillas (Templates) ▼
# Listar plantillas GET /api/templates # Obtener detalles de plantilla GET /api/templates/{id} # Crear plantilla POST /api/templates { "name": "Ubuntu 22.04", "description": "Plantilla base de Ubuntu", "os": "linux" } # Actualizar plantilla PUT /api/templates/{id} # Eliminar plantilla DELETE /api/templates/{id}
Usuarios (Users) ▼
# Listar usuarios GET /api/users # Obtener detalles de usuario GET /api/users/{id} # Crear usuario POST /api/users { "username": "nuevo-usuario", "email": "usuario@example.com", "password": "secure-password", "role": "user" } # Actualizar usuario PUT /api/users/{id} # Eliminar usuario DELETE /api/users/{id}
Hipervisores (Hypervisors) ▼
# Listar hipervisores GET /api/hypervisors # Obtener estado de hipervisor GET /api/hypervisors/{id} # Añadir hipervisor POST /api/hypervisors { "name": "KVM-Server-1", "type": "kvm", "host": "192.168.1.100", "port": 16509 } # Eliminar hipervisor DELETE /api/hypervisors/{id}

Ejemplo de Integración con Python

import requests # Configuración BASE_URL = "https://isardvdi.local/api" USERNAME = "admin" PASSWORD = "your-password" # Obtener token def get_token(): response = requests.post( f"{BASE_URL}/auth/token", json={"username": USERNAME, "password": PASSWORD} ) return response.json()["access"] # Crear escritorio def create_desktop(token, name, template): headers = {"Authorization": f"Bearer {token}"} data = { "name": name, "template": template, "ram": 2048, "vcpus": 2 } response = requests.post( f"{BASE_URL}/desktops", headers=headers, json=data ) return response.json() # Uso token = get_token() desktop = create_desktop(token, "Mi Escritorio", "ubuntu-22.04") print(f"Escritorio creado: {desktop['id']}")

Troubleshooting

Solución de problemas comunes

Problemas Comunes

No puedo acceder a la interfaz web ▼

Causas posibles:

  • Los contenedores no están iniciados
  • El puerto 443 está bloqueado por el firewall
  • Problemas con el certificado SSL

Soluciones:

# Verificar estado de contenedores docker-compose ps # Ver logs docker-compose logs -f # Reiniciar contenedores docker-compose restart # Verificar puertos sudo netstat -tlnp | grep -E ':(80|443)' # Abrir puertos en firewall sudo ufw allow 80/tcp sudo ufw allow 443/tcp
Los escritorios no se inician ▼

Causas posibles:

  • El hipervisor no está configurado correctamente
  • No hay suficientes recursos disponibles
  • Problemas con la plantilla

Soluciones:

# Verificar estado del hipervisor virsh nodeinfo # Verificar recursos disponibles virsh freemem --all virsh vcpuinfo --all # Ver logs de libvirt sudo journalctl -u libvirtd -f # Reiniciar libvirt sudo systemctl restart libvirtd
Conexión VNC falla ▼

Causas posibles:

  • Puertos VNC bloqueados
  • Problemas con WebSocket
  • Configuración de proxy inverso incorrecta

Soluciones:

# Verificar puertos VNC sudo netstat -tlnp | grep 59 # Abrir rango de puertos VNC sudo ufw allow 5900:6000/tcp # Verificar configuración de nginx sudo nginx -t sudo systemctl restart nginx
Autenticación LDAP falla ▼

Causas posibles:

  • Credenciales incorrectas
  • Conectividad con el servidor LDAP
  • Configuración de DN incorrecta

Soluciones:

# Probar conexión LDAP ldapsearch -x -H ldap://ad.example.com -D "CN=admin,DC=example,DC=com" -W # Verificar configuración python manage.py test_ldap # Ver logs de autenticación docker-compose logs -f isardvdi | grep -i ldap
Rendimiento lento de escritorios ▼

Causas posibles:

  • Recursos insuficientes
  • Almacenamiento lento
  • Problemas de red

Soluciones:

# Verificar uso de recursos htop iostat -x 1 # Optimizar almacenamiento # Usar SSD/NVMe para almacenamiento de VMs # Aumentar recursos de escritorio # Editar configuración de plantilla

Herramientas de Diagnóstico

Use estas herramientas para diagnosticar problemas:

Logs del Sistema

# Logs de Isard VDI docker-compose logs -f # Logs de libvirt sudo journalctl -u libvirtd -f # Logs de nginx sudo tail -f /var/log/nginx/error.log

Diagnóstico de Red

# Verificar conectividad ping isardvdi.local # Verificar puertos nmap -p 80,443,5900-6000 isardvdi.local # Verificar WebSocket wscat -c wss://isardvdi.local/ws/desktop/1

Preguntas Frecuentes (FAQ)

Respuestas a las preguntas más comunes

¿Isard VDI es gratuito? ▼

Sí, Isard VDI es completamente gratuito y de código abierto bajo la licencia AGPL-3.0. Puede usarlo, modificarlo y distribuirlo libremente.

¿Cuántos usuarios concurrentes soporta? ▼

El número de usuarios concurrentes depende de los recursos de su servidor. Con un servidor adecuado (16+ GB RAM, 8+ vCPUs), puede soportar 50-100 usuarios concurrentes. Para más usuarios, considere usar múltiples hipervisores o Kubernetes.

¿Puedo usar Isard VDI en la nube? ▼

Sí, Isard VDI puede desplegarse en cualquier proveedor de nube que soporte Docker o Kubernetes, como AWS, Google Cloud, Azure, DigitalOcean, etc.

¿Qué sistemas operativos soporta como escritorios? ▼

Isard VDI soporta cualquier sistema operativo que pueda virtualizarse, incluyendo Windows (7, 10, 11, Server), Linux (Ubuntu, Debian, CentOS, Fedora, etc.), y macOS (con ciertas limitaciones).

¿Cómo realizo backup de los escritorios? ▼

Puede realizar backup de los escritorios usando snapshots de libvirt o copiando los archivos de disco de las VMs. También puede configurar backups automáticos usando herramientas como rsync o soluciones de backup de nube.

¿Puedo integrar Isard VDI con mi sistema de gestión de usuarios existente? ▼

Sí, Isard VDI soporta integración con LDAP, Active Directory, y OAuth2. También puede usar la API REST para integración con sistemas personalizados.

¿Hay soporte técnico disponible? ▼

Como proyecto de código abierto, el soporte comunitario está disponible a través de GitLab Issues y foros. También puede contratar soporte comercial a través de los desarrolladores del proyecto.

¿Cómo actualizo Isard VDI a la última versión? ▼

Para actualizar, simplemente ejecute:

# Actualizar el repositorio git pull # Reconstruir y reiniciar contenedores docker-compose down docker-compose pull docker-compose up -d
