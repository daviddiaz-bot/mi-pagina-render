1. Introducción y Requisitos

¿Qué es n8n?

n8n es una plataforma de automatización de workflows open source que permite conectar diferentes servicios y APIs sin necesidad de código. Es ideal para ofrecer servicios de automatización a clientes.

✓ Self-hosted
Control total de datos
✓ Sin límites
Ejecuciones ilimitadas
✓ 400+ nodos
Integraciones variadas
✓ Extensible
Nodos personalizados

📋 Requisitos Previos

  • ✅ Cuenta en Render.com
  • ✅ Cuenta en Supabase (base de datos)
  • ✅ Dominio personalizado (opcional pero recomendado)
  • ✅ Conocimientos básicos de Docker
  • ✅ Git instalado en tu máquina
⚠️ Importante: El plan gratuito de Render tiene limitaciones. Para producción, se recomienda el plan Starter ($7/mes) para mejor rendimiento y disponibilidad.

3. Preparación del Entorno

1 Crear cuenta en Render

1. Ve a render.com

2. Haz clic en "Get Started for Free"

3. Regístrate con GitHub, GitLab o email

4. Verifica tu cuenta

2 Crear cuenta en Supabase

1. Ve a supabase.com

2. Haz clic en "Start your project"

3. Crea una organización

4. Crea un nuevo proyecto

5. Guarda la contraseña de la base de datos

# Obtener credenciales de Supabase # Settings > Database > Connection string (URI) # Formato de conexión: postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres
3 Preparar archivo Docker Compose

Crea un archivo docker-compose.yml para n8n:

version: '3.8' services: n8n: image: n8nio/n8n:latest ports: - "5678:5678" environment: - N8N_HOST=${N8N_HOST} - N8N_PORT=5678 - N8N_PROTOCOL=https - NODE_ENV=production - WEBHOOK_URL=${WEBHOOK_URL} - GENERIC_TIMEZONE=America/Bogota - DB_TYPE=postgresdb - DB_POSTGRESDB_HOST=${DB_HOST} - DB_POSTGRESDB_PORT=6543 - DB_POSTGRESDB_DATABASE=postgres - DB_POSTGRESDB_USER=${DB_USER} - DB_POSTGRESDB_PASSWORD=${DB_PASSWORD} - N8N_BASIC_AUTH_ACTIVE=true - N8N_BASIC_AUTH_USER=${N8N_USER} - N8N_BASIC_AUTH_PASSWORD=${N8N_PASSWORD} volumes: - n8n_data:/home/node/.n8n restart: always volumes: n8n_data:

4. Configuración en Render

4 Crear Web Service en Render

Opción A: Usando imagen Docker

  1. En el dashboard de Render, haz clic en "New +"
  2. Selecciona "Web Service"
  3. Elige "Deploy a Docker image"
  4. Ingresa la imagen: n8nio/n8n:latest
  5. Configura el nombre del servicio (ej: mi-n8n)
  6. Selecciona la región más cercana
  7. Elige el plan (Free o Starter)
💡 Tip: El plan Free tiene limitaciones: 750 horas/mes, se suspende después de 15 min de inactividad. Para producción, usa Starter ($7/mes).
5 Configurar Variables de Entorno

Variables de entorno requeridas en Render:

# Variables de entorno para Render # Host y URL N8N_HOST=tu-n8n.onrender.com WEBHOOK_URL=https://tu-n8n.onrender.com/ # Autenticación N8N_BASIC_AUTH_ACTIVE=true N8N_BASIC_AUTH_USER=admin N8N_BASIC_AUTH_PASSWORD=tu-contraseña-segura # Base de datos (Supabase) DB_TYPE=postgresdb DB_POSTGRESDB_HOST=aws-0-us-east-1.pooler.supabase.com DB_POSTGRESDB_PORT=6543 DB_POSTGRESDB_DATABASE=postgres DB_POSTGRESDB_USER=postgres.tu-project-ref DB_POSTGRESDB_PASSWORD=tu-contraseña-supabase # Configuración NODE_ENV=production GENERIC_TIMEZONE=America/Bogota N8N_PORT=5678
⚠️ Seguridad: Nunca uses contraseñas débiles. Genera contraseñas seguras con: openssl rand -base64 32

5. Despliegue de n8n

6 Desplegar desde Docker Hub

Configuración en Render Dashboard:

# Configuración del servicio Web Name: n8n-production Region: Oregon (us-west) o Virginia (us-east) Service Type: Web Service Environment: Docker Image URL: n8nio/n8n:latest Plan: Starter ($7/mes) recomendado # Puerto Port: 5678 # Comando de inicio (opcional) Start Command: n8n start
7 Desplegar desde Repositorio Git

Alternativa: Crear repositorio propio

# Estructura del repositorio n8n-deploy/ ├── Dockerfile ├── docker-compose.yml ├── .env.example └── README.md # Dockerfile personalizado FROM n8nio/n8n:latest # Instalar nodos personalizados si es necesario # RUN npm install n8n-nodes-custom # Configurar permisos USER node # Exponer puerto EXPOSE 5678 # Comando de inicio CMD ["n8n", "start"]

Archivo .env.example:

# Copiar a .env y completar valores N8N_HOST=tu-dominio.com WEBHOOK_URL=https://tu-dominio.com/ N8N_BASIC_AUTH_USER=admin N8N_BASIC_AUTH_PASSWORD=cambiar-esto DB_POSTGRESDB_HOST=tu-host.supabase.com DB_POSTGRESDB_USER=postgres.ref DB_POSTGRESDB_PASSWORD=tu-contraseña
✅ Verificación: Después del despliegue, accede a https://tu-servicio.onrender.com para verificar que n8n está funcionando.

6. Configuración de Base de Datos

Configurar PostgreSQL en Supabase

8 Obtener credenciales de Supabase
  1. Ve a tu proyecto en Supabase
  2. Settings > Database
  3. Copia la "Connection string" (URI)
  4. Usa el puerto 6543 para conexiones pooler (recomendado)
# Connection string de Supabase # Formato: postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres # Ejemplo: postgresql://postgres.abc123:MiPassword123@aws-0-us-east-1.pooler.supabase.com:6543/postgres # Variables para Render: DB_POSTGRESDB_HOST=aws-0-us-east-1.pooler.supabase.com DB_POSTGRESDB_PORT=6543 DB_POSTGRESDB_DATABASE=postgres DB_POSTGRESDB_USER=postgres.abc123 DB_POSTGRESDB_PASSWORD=MiPassword123

Crear tablas para n8n (Opcional)

Si necesitas tablas personalizadas para tus workflows:

-- Ejecutar en Supabase SQL Editor -- Tabla para logs de workflows CREATE TABLE workflow_logs ( id SERIAL PRIMARY KEY, workflow_id VARCHAR(255), execution_id VARCHAR(255), status VARCHAR(50), created_at TIMESTAMP DEFAULT NOW(), data JSONB ); -- Tabla para configuraciones de clientes CREATE TABLE client_configs ( id SERIAL PRIMARY KEY, client_name VARCHAR(255), webhook_url VARCHAR(500), api_key VARCHAR(255), active BOOLEAN DEFAULT true, created_at TIMESTAMP DEFAULT NOW() ); -- Índices para mejor rendimiento CREATE INDEX idx_workflow_logs_workflow ON workflow_logs(workflow_id); CREATE INDEX idx_workflow_logs_status ON workflow_logs(status);

7. Dominio Personalizado y SSL

9 Configurar dominio personalizado
  1. En Render, ve a tu servicio n8n
  2. Settings > Custom Domain
  3. Agrega tu dominio (ej: n8n.tuempresa.com)
  4. Configura los DNS según las instrucciones:
# Configuración DNS (ejemplo con Cloudflare) Tipo: CNAME Nombre: n8n Contenido: tu-servicio.onrender.com Proxy: Activado (para SSL automático) # O con tu proveedor de dominios: Tipo: CNAME Host: n8n Valor: tu-servicio.onrender.com TTL: 3600
✅ SSL Automático: Render proporciona certificados SSL gratuitos automáticamente con Let's Encrypt.

9. Monitoreo y Mantenimiento

📊 Métricas a Monitorear

  • ✅ Uso de CPU y memoria
  • ✅ Tiempo de respuesta de webhooks
  • ✅ Errores en ejecuciones
  • ✅ Conexiones a base de datos
  • ✅ Almacenamiento utilizado
# Configurar alertas en Render # Settings > Notifications # Alertas recomendadas: - CPU > 80% por 5 minutos - Memory > 85% por 5 minutos - Service unhealthy - Deploy failed

🔄 Backups Automáticos

# Script de backup (ejecutar en cron) #!/bin/bash # backup-n8n.sh DATE=$(date +%Y%m%d_%H%M%S) BACKUP_DIR="/backups/n8n" # Exportar workflows curl -X GET "https://tu-n8n.onrender.com/api/v1/workflows" \ -H "X-N8N-API-KEY: tu-api-key" \ -o "$BACKUP_DIR/workflows_$DATE.json" # Backup de base de datos (Supabase) pg_dump $DATABASE_URL > "$BACKUP_DIR/db_$DATE.sql" # Limpiar backups antiguos (más de 30 días) find $BACKUP_DIR -type f -mtime +30 -delete

10. Escalado y Optimización

📈 Planes de Escalado

Fase Plan Costo Especificaciones
Inicio Free $0/mes 512MB RAM, se suspende
Producción Starter $7/mes 512MB RAM, siempre activo
Crecimiento Standard $25/mes 2GB RAM, mejor rendimiento
Enterprise Pro $85/mes 4GB RAM, múltiples instancias

⚡ Optimizaciones de Rendimiento

# Variables de entorno para optimización # Aumentar timeout para workflows largos N8N_DEFAULT_TIMEOUT=300000 # Habilitar queue para alta demanda QUEUE_BULL_REDIS_HOST=redis-host QUEUE_BULL_REDIS_PORT=6379 # Configurar workers EXECUTIONS_MODE=queue EXECUTIONS_WORKERS=3 # Cache de nodos N8N_CACHE_ENABLED=true N8N_CACHE_TTL=3600

12. Mejores Prácticas

✅ Checklist de Producción

  • ☑️ Autenticación básica habilitada
  • ☑️ SSL/HTTPS configurado
  • ☑️ Base de datos externa (Supabase)
  • ☑️ Backups automáticos configurados
  • ☑️ Monitoreo y alertas activos
  • ☑️ Variables de entorno seguras
  • ☑️ Dominio personalizado
  • ☑️ Rate limiting configurado
  • ☑️ Logs centralizados
  • ☑️ Documentación de workflows

🔒 Seguridad

⚠️ Nunca hagas esto:
  • ❌ Usar contraseñas débiles
  • ❌ Exponer API keys en código
  • ❌ Desactivar autenticación
  • ❌ Ignorar actualizaciones de seguridad
✅ Siempre haz esto:
  • ✅ Usar variables de entorno
  • ✅ Rotar credenciales regularmente
  • ✅ Limitar acceso por IP si es posible
  • ✅ Mantener n8n actualizado
